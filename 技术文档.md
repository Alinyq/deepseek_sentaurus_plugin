## 1. 项目概述

### 1.1 项目定位
TCAD AI 助手是一款基于 LLM（大语言模型）的 Sentaurus TCAD 仿真辅助工具，集成在 Synopsys Sentaurus Workbench 中，为半导体器件仿真工程师提供：
- **AI 智能对话** — 通过 Agent 自动调用 TCAD 工具获取真实数据、管理实验、分析结果
- **代码审查** — 自动分析 .cmd 命令文件的语法、物理模型、网格、收敛等问题
- **结果分析** — 解析仿真结果并给出优化建议
- **参数优化** — 基于现有仿真数据给出参数优化方向
- **报告生成** — 自动生成仿真报告

### 1.2 技术栈
| 组件 | 技术 |
|------|------|
| GUI 框架 | PyQt6 |
| AI 模型 | DeepSeek Chat（兼容 OpenAI API） |
| Agent 框架 | Upsonic（支持 Agent + Tools + Skills） |
| TCAD 交互 | gpythonsh + swbpy2（SWB Python API） |
| Markdown 渲染 | markdown 库 + QTextBrowser HTML |
| 流式输出 | SSE（Server-Sent Events）+ QThread |

## 4. 核心模块详解

### 4.1 main.py — GUI 主入口

**类：`DeepSeekTCADGUI(QMainWindow)`**

主窗口类，继承自 `QMainWindow`，管理所有标签页和全局状态。

**核心属性：**
- `project_path` — 当前项目路径（全局共享）
- `client` — `DeepSeekClient` 实例（用于功能标签页的HTTP请求）
- `project_parser` — `TCADProjectParser` 实例（解析SWB项目）
- `active_threads` — 活跃的 MarkdownThread 列表（用于停止控制）

**初始化流程：**
```
DeepSeekTCADGUI(project_path)
  ├── load_config()                    # 读取 settings.ini
  ├── DeepSeekClient(api_key, model)   # 初始化HTTP客户端
  ├── TCADFileReader()                 # 初始化文件读取器
  ├── TCADProjectParser(project_path)  # 初始化项目解析器
  └── setup_ui()
        ├── 项目路径选择栏
        ├── QTabWidget（5个标签页）
        └── 状态栏
```

**关键方法：**
| 方法 | 功能 |
|------|------|
| `browse_project()` | 选择项目目录，同步更新所有组件 |
| `_run_query()` | 通用AI查询方法，支持流式输出和Markdown渲染 |
| `_stop_query()` | 停止当前AI查询 |

### 4.2 DeepSeekClient — API 客户端

**类：`DeepSeekClient`**

封装 DeepSeek/兼容 OpenAI API 的 HTTP 请求。

**核心方法：**
| 方法 | 说明 | 返回方式 |
|------|------|----------|
| `chat_completion(messages)` | 非流式一次性返回 | 完整文本 |
| `stream_chat(messages)` | 流式逐token返回 | generator (yield) |

**流式实现：**
```python
def stream_chat(self, messages):
    response = requests.post(url, json={"stream": True, ...}, stream=True)
    for line in response.iter_lines():
        if line.startswith('data: '):
            delta = json.loads(line[6:])['choices'][0]['delta']
            yield delta.get('content', '')
```

### 4.3 ChatWidget — AI 对话 UI

**类：`ChatWidget(QWidget)`**

独立于 main.py 的聊天组件，通过 `main.py` 的 `create_chat_tab()` 创建。

**UI 结构：**
```
ChatWidget
├── Header（标题 + 清空按钮）
├── QScrollArea（消息滚动区）
│   └── msg_container (QWidget)
│       ├── ChatMessageBubble × N
│       └── stretch (占位符)
├── Input Frame
│   ├── QLineEdit（输入框，Enter发送）
│   └── QPushButton（发送按钮）
└── Status Bar（状态提示）
```

**关键设计：**
- **ChatMessageBubble** — 自定义消息气泡组件
  - 用户消息：绿色背景 (#dcf8c6)，左对齐
  - 助手消息：白色背景 + 边框，右对齐
  - 支持 Markdown 渲染（通过 `markdown` 库转 HTML）
  - 支持 `append_text()` 实时追加 token（流式输出）

### 4.4 UpsonicClient — Agent 客户端

**类：`UpsonicClient(QObject)`**

基于 Upsonic Agent 框架的 AI 客户端，支持：
- Agent 自动工具调用
- 多轮对话记忆
- Skills 渐进式加载
- 流式输出

**核心组件：**

```
UpsonicClient
├── Agent（Upsonic智能代理）
│   ├── OpenAIChatModel（DeepSeek模型）
│   ├── system_prompt（行为指南）
│   ├── Skills（sentaurus-tcad技能包）
│   ├── internal messages（对话历史记忆）
│   └── tools（17个TCAD工具）
│
├── AgentWorker（QThread工作对象）
│   ├── _async_run() — 异步执行 Agent.astream()
│   ├── run() — 在独立事件循环中执行
│   └── 信号：token_received / response_complete / error_occurred
│
└── ConversationState
    └── all_tools — 工具列表缓存
```

**线程模型：**
```
GUI主线程 ──发送消息──▶ QThread
                           │
                    AgentWorker.run()
                           │
                    asyncio.new_event_loop()
                           │
                    agent.astream(task)
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
     token_received   response_complete  error_occurred
          │                │                │
          └────────────────┴────────────────┘
                           │（信号槽机制）
                           ▼
                    ChatWidget.on_token()
                    ChatWidget.on_complete()
                    ChatWidget.on_error()
```

## 5. AI 对话 Agent 详解

### 5.1 Agent 配置

```python
Agent(
    model=OpenAIChatModel(
        model_name="deepseek-chat",
        provider=OpenAIProvider(
            api_key=api_key,
            base_url="https://api.deepseek.com"
        )
    ),
    name="TCAD Assistant",
    instructions=system_prompt,   # 系统提示词
    skills=Skills(loaders=[LocalSkills(skills_path)]),  # 技能包
    debug=False
)
```

### 5.2 System Prompt（系统提示词）

Agent 的行为指南，包含三个部分：

1. **角色定义** — TCAD 仿真专家助手
2. **工具说明** — 拥有 TCAD 相关工具
3. **知识技能库** — 告知 Agent 拥有 sentaurus-tcad 技能包
4. **行为指南**（7条规则）：
   - 智能选择是否调用工具
   - 涉及真实数据时先调用工具
   - 已有足够信息时不再重复调用
   - 利用对话记忆避免重复调用
   - 不凭猜测回答具体数据问题
   - 用中文回复
   - 代码和命令使用 Markdown 格式

### 5.3 多轮对话记忆

Upsonic Agent 内部维护 `messages` 列表，自动记录：
- System Prompt（初始注入）
- 用户消息（每次 send_message）
- 助手回复（完整响应）
- Tool Call 和 Tool Result（工具调用及结果）

**清空对话逻辑：**
```python
def clear_history(self):
    self.state.clear()           # 清空工具列表缓存
    self._create_agent()         # 重新创建 Agent（清空 messages）
    self.add_tools(self.state.all_tools)  # 重新添加工具
```

### 5.4 Skills 技能系统

**配置方式：**
```python
skills_path = os.path.join(plugin_dir, 'skills', 'sentaurus-tcad')
skills = Skills(loaders=[LocalSkills(skills_path)])
agent = Agent(..., skills=skills)
```

**渐进式披露机制：**
1. Agent 启动时，仅加载 SKILL.md 的 name + description
2. Agent 需要时，调用 `get_skill_instructions()` 加载完整指令
3. Agent 需要时，调用 `get_skill_reference()` 加载参考文档
4. Agent 需要时，调用 `get_skill_script()` 加载脚本文件

## 6. Agent Tools 工具列表

共 **17 个** TCAD 工具，全部通过 `@tool` 装饰器定义在 `tcad_tools.py` 中，底层通过 `gpythonsh` 执行 `tcad_ai_tools.py` 脚本来操作 SWB 项目。

|---|--------|------|----------|

| 2 | `get_experiment_list` | 无 | 获取所有实验列表（索引、参数值） |
| 3 | `get_cmd_files` | 无 | 获取所有 .cmd 命令文件内容 |
| 4 | `get_param_names` | 无 | 获取所有参数（自变量）名称 |
| 5 | `get_var_names` | 无 | 获取所有变量（因变量/输出量）名称 |
| 6 | `get_param_value` | `experiment_index`, `param_name` | 获取指定实验的特定参数值 |
| 7 | `set_param_value` | `experiment_index`, `param_name`, `value` | 修改指定实验的特定参数值 |
| 8 | `get_experiment_status` | 无 | 获取所有实验状态（等待中/运行中/完成/错误） |
| 9 | `run_experiment` | `experiment_index` | 运行指定单个实验 |
| 10 | `run_all_experiments` | 无 | 运行所有实验 |
| 11 | `add_experiment` | `name`, `parameter_values`(JSON) | 添加新实验 |
| 12 | `delete_experiment` | `experiment_index` | 删除指定实验 |
| 13 | `check_errors` | 无 | 检查所有实验的错误信息 |
| 14 | `get_node_list` | 无 | 获取所有节点ID列表 |
| 15 | `read_file` | `file_path` | 读取指定文件内容 |
| 16 | `write_file` | `file_path`, `content` | 写入内容到指定文件 |
| 17 | `list_files` | `dir_path` | 列出目录内容 |

## 8. 功能标签页详解

### 8.1 代码审查（Code Review）

**功能**：分析 Sentaurus .cmd 命令文件，检查语法、物理模型、网格、收敛等问题。

**UI 组件**：
- 文件路径输入 + 浏览按钮（选择 .cmd 文件）
- 补充说明输入框（额外需求）
- 结果展示区（Markdown 渲染）
- 代码审查按钮（点击后切换为"停止"）

**数据流**：
```
1. 用户选择 .cmd 文件
2. 读取文件内容
3. 构建 messages（system + user）
4. _run_query() → DeepSeekClient.stream_chat()
5. 逐 token 接收 → MarkdownRenderer.render() → QTextBrowser
```

**Prompt 模板**：
```
Please review this Sentaurus TCAD code file (xxx.cmd):

```tcl
{code_content}
```

Check for:
1. Syntax errors
2. Physical model selection
3. Mesh settings
4. Solver parameters
5. Boundary conditions
6. Convergence issues
```

### 8.2 结果分析（Result Analysis）

**功能**：解析仿真结果并给出性能分析和优化建议。

**UI 组件**：
- 结果文件路径输入 + 浏览按钮（.plt/.log/.cmd/.tdr）
- 自动检测结果文件按钮
- 补充说明输入框
- 结果展示区
- 分析结果按钮

**两种使用方式**：
1. **浏览选择** → 选择特定结果文件 → 读取内容 → 分析
2. **自动检测** → 通过 TCADProjectParser.parse_results() 获取完整项目信息 → 分析

**数据流**：
```
自动检测模式：
1. auto_detect_results()
2. project_parser.parse_results()
   ├── gpythonsh get_swb_info.py（full模式）
   │   ├── 工具节点列表
   │   ├── 参数列表
   │   ├── 结果变量
   │   ├── cmd文件内容
   │   ├── gtree.dat 内容
   │   └── gtooldb.tcl 内容
   └── 格式化输出
3. 显示到 result_browser
4. analyze_results() → _run_query() → 流式输出
```

### 8.3 参数优化（Parameter Optimization）

**功能**：基于现有仿真数据，给出参数优化方向建议。

**UI 组件**：
- 目标性能输入框
- 可调参数输入框
- 结果展示区
- 开始优化按钮

**数据流**：
```
1. 用户填写目标性能和可调参数
2. project_parser.parse_all() 获取完整项目信息
3. 构建 messages → _run_query() → 流式输出
```

### 8.4 报告生成（Report Generation）

**功能**：自动生成综合性的 TCAD 仿真报告。

**UI 组件**：
- 报告标题输入框
- 补充说明输入框
- 结果展示区
- 生成报告按钮
- 保存报告按钮（.md / .txt）

**数据流**：
```
1. 用户输入报告标题
2. project_parser.parse_all() 获取项目摘要
3. 构建 messages → _run_query() → 流式输出
4. 用户可点击"保存报告"导出为 Markdown 文件
```

### 8.5 AI 对话（Chat）

**功能**：智能对话，Agent 自动调用工具获取真实数据，支持多轮对话。

**UI 组件**：
- 标题栏（AI对话 + 清空按钮）
- 消息滚动区（气泡式布局）
- 输入框 + 发送按钮
- 状态栏

**数据流**：
```
1. 用户输入消息
2. ChatWidget.send() → UpsonicClient.send_message()
3. 创建 Task(description=message)
4. 创建 AgentWorker + QThread
5. agent.astream(task) → 流式输出
   ├── Agent 决定调用工具
   ├── 执行工具 → 获取结果
   ├── 生成回复 → token_received 信号
   └── ChatWidget.on_token() → 气泡实时更新
6. response_complete → ChatWidget.on_complete() → 气泡定稿

## 10. 关键设计模式

### 10.1 信号槽（Qt Signal-Slot）

用于跨线程通信：
```
QThread ──started──▶ Worker.run()
Worker ──token_received──▶ ChatWidget.on_token()
Worker ──response_complete──▶ ChatWidget.on_complete()
Worker ──response_complete──▶ QThread.quit()
QThread ──finished──▶ worker.deleteLater()
```

### 10.2 异步事件循环隔离

Agent 在独立 asyncio 事件循环中运行，避免阻塞 Qt 主线程：
```python
def run(self):
    loop = asyncio.new_event_loop()
    asyncio.set_event_loop(loop)
    loop.run_until_complete(self._async_run())
    loop.close()
```

### 10.3 流式 Markdown 渲染

```
token 流入 ──▶ ChatMessageBubble.append_text()
                └─▶ markdown.markdown(raw_text)
                    └─▶ QLabel.setText(html)
```

每收到一个 token 就重新渲染整个文本（因为 Markdown 是整体解析的），通过缓存 `_raw_text` 实现追加。

### 10.4 气泡高度自适应

```python
# ChatMessageBubble:
layout = QVBoxLayout(self)
layout.addWidget(self.label)  # QLabel，wordWrap=True
# 没有 setFixedHeight，让气泡高度由内容决定
```

### 10.5 多轮对话记忆

Upsonic Agent 内部维护 `messages` 列表：
- 初始：注入 system prompt
- 每次发送：追加 user message
- 每次接收：追加 assistant message
- 工具调用：追加 tool call 和 tool result

清空对话时重新创建 Agent 实例。
