# DeepSeek TCAD Plugin - 安装与使用说明书

## 一、项目架构

本项目采用**插件与Sentaurus项目分离**的架构设计：

### 1.1 插件目录（可移植）

插件代码是独立的、可复用的，可以复制到任何Sentaurus机器上安装使用。

```
/home/tcad/work/deepseek_tcad_plugin/
├── src/                                   # 插件源码
│   └── main.py                            # 主程序入口（GUI）
├── core/                                  # 核心模块
│   ├── __init__.py
│   └── deepseek_client.py                 # DeepSeek API客户端
├── tcad/                                  # TCAD模块
│   ├── __init__.py
│   ├── file_reader.py                     # 文件读取器
│   └── project_parser.py                  # 项目解析器
├── config/
│   └── settings.ini                       # API Key配置
├── icons/
│   └── deepseek.gif                       # 工具图标（GIF格式，64x64）
├── swb_tools/
│   ├── launchers/
│   │   └── launch_deepseek.sh             # 启动脚本
│   └── deepseek_tcad_ai.py               # 节点启动器
├── install.sh                             # 一键安装脚本
└── .venv/                                 # Python虚拟环境（由install.sh自动创建）
```

### 1.2 Sentaurus项目目录

Sentaurus项目只包含TCAD仿真相关文件。安装插件后，项目中会多一个 `deepseek_tcad_ai.py` 启动器文件。

```
/path/to/your/swb/project/
├── deepseek_tcad_ai.py                    # 节点启动器（由安装脚本自动创建）
├── gtree.dat                              # SWB项目树
├── PNdiode_dvs.cmd                        # SDE命令文件
├── IV_des.cmd                             # SDEVICE命令文件
├── n*.tdr, n*.plt, n*.log                 # 仿真结果
└── ...                                    # 其他TCAD文件
```

| 项目 | 路径 | 说明 |

| 插件目录 | `/home/tcad/work/deepseek_tcad_plugin/` | 所有插件代码 |
| Python环境 | `/home/tcad/work/deepseek_tcad_plugin/.venv/bin/python` | uv管理的Python 3.14.2 |

| 节点启动器 | `<项目目录>/deepseek_tcad_ai.py` | 每个SWB项目一个 |

### 2.2 模块导入路径说明（重要！）

`src/main.py` 中使用以下代码自动定位插件根目录：

```python
PLUGIN_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
sys.path.insert(0, PLUGIN_DIR)
```

- `__file__` = `/home/tcad/work/deepseek_tcad_plugin/src/main.py`
- `os.path.dirname(__file__)` = `src/`
- 再 `dirname` 一次 = 插件根目录（即 `deepseek_tcad_plugin/`）

因此 `core/`、`tcad/` 等模块必须在**插件根目录**下，不能在 `src/` 下。

## 三、在新机器上安装插件

### 3.1 前置条件

1. **Sentaurus Workbench 已安装**（W-2024.09或更高版本）
   - 确认方法：运行 `swb` 能启动
   - 记录Sentaurus安装路径，通常是 `/usr/synopsys/sentaurus/W-2024.09/`

2. **uv 已安装**（Python包管理器）
   - 确认方法：运行 `uv --version`
   - 如果没有，安装：`curl -LsSf https://astral.sh/uv/install.sh | sh`

3. **sudo 权限**（用于复制图标到系统目录）
   - 图标需要放到 `/usr/synopsys/sentaurus/*/tcad/current/lib/glib2/icons/`

4. **DeepSeek API Key**
   - 从 https://platform.deepseek.com 获取

### 3.2 安装步骤

**方式一：使用一键安装脚本（推荐）**

1. 将整个 `deepseek_tcad_plugin` 文件夹复制到目标机器：
   ```bash
   # 确保插件目录在正确位置
   cp -r deepseek_tcad_plugin /home/tcad/work/
   ```

2. 运行安装脚本：
   ```bash
   cd /home/tcad/work/deepseek_tcad_plugin
   bash install.sh
   ```

   脚本会自动完成：
   - 创建 Python 虚拟环境（使用 uv）
   - 安装依赖（requests, pillow）
   - 注册 SWB 工具（写入 tooldb_tcad）
   - 安装图标到 SWB 图标目录
   - 配置 API Key（交互式输入）

**方式二：手动安装**

```bash
# === 1. 设置插件路径 ===
PLUGIN_DIR="/home/tcad/work/deepseek_tcad_plugin"

# === 2. 创建Python虚拟环境 ===
cd "${PLUGIN_DIR}"
uv venv .venv --python 3.14 --python-preference managed
source .venv/bin/activate
pip install requests pillow

# === 3. 确认Python路径 ===
PYTHON_PATH="${PLUGIN_DIR}/.venv/bin/python"
${PYTHON_PATH} --version  # 应该显示 Python 3.14.x

# === 4. 配置API Key ===
nano "${PLUGIN_DIR}/config/settings.ini"
# 修改 api_key = sk-你的真实APIKey

# === 5. 安装图标 ===
# 确保有GIF格式的图标（64x64）
sudo cp "${PLUGIN_DIR}/icons/deepseek.gif" \
        /usr/synopsys/sentaurus/W-2024.09/tcad/current/lib/glib2/icons/deepseek.gif
# 注意：如果你的Sentaurus版本路径不同，需要相应调整

# === 6. 注册SWB工具 ===
# 创建用户工具目录
mkdir -p /home/tcad/STDB

# 写入工具定义
cat > /home/tcad/STDB/tooldb_tcad << EOF
# tool: deepseek_tcad
set WB_tool(deepseek_tcad,category) utility
set WB_tool(deepseek_tcad,visual_category) utility
set WB_tool(deepseek_tcad,acronym) ai
set WB_tool(deepseek_tcad,after) all
set WB_binaries(tool,deepseek_tcad) ${PYTHON_PATH}
set Icon(deepseek_tcad) \$app_data(icon_dir)/deepseek.gif
set WB_tool(deepseek_tcad,setup) { os_ln_rel @commands@ n@node@_ai.py "@pwdout@/@nodedir@" }
set WB_tool(deepseek_tcad,epilogue) { extract_vars "\$nodedir" @stdout@ @node@ }
set WB_tool(deepseek_tcad,cmd_line) "n@node@_ai.py"
set WB_tool(deepseek_tcad,input) [list commands pref]
set WB_tool(deepseek_tcad,input,commands,label)  "DeepSeek AI Commands..."
set WB_tool(deepseek_tcad,input,commands,editor)  text
set WB_tool(deepseek_tcad,input,commands,file)   @tool_label@_ai.py
set WB_tool(deepseek_tcad,input,pref,file)  @tool_label@_ai.prf
set WB_tool(deepseek_tcad,input,pref,label)  "Preferences..."
set WB_tool(deepseek_tcad,input,pref,editor)  pref
set WB_tool(deepseek_tcad,output) [list]
set WB_tool(deepseek_tcad,output,files) "n@node@_* pp@node@_* *_n@node@_*"
set WB_tool(deepseek_tcad,exec_dependency) strict
set WB_tool(deepseek_tcad,available) { return 1 }
lappend WB_tool(all) deepseek_tcad
EOF

# === 7. 在你的Sentaurus项目中创建节点启动器 ===
# 进入你的SWB项目目录
PROJECT_DIR="/path/to/your/swb/project"

cat > "${PROJECT_DIR}/deepseek_tcad_ai.py" << EOF
#!/${PYTHON_PATH}
import sys, os, subprocess
subprocess.Popen([sys.executable, "${PLUGIN_DIR}/src/main.py", "--project", os.getcwd()],
                 stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL, start_new_session=True)
print("DeepSeek AI Assistant launched!")
EOF
chmod +x "${PROJECT_DIR}/deepseek_tcad_ai.py"
```

### 3.3 注意事项（容易踩坑的地方）

1. **Python路径必须是插件目录下的 `.venv/bin/python`**
   - 不要使用系统 python3，它可能没有 tkinter 或 requests
   - 不要 symlink 到其他位置的 Python

2. **`core/` 和 `tcad/` 目录必须在插件根目录下**
   - 目录结构：`deepseek_tcad_plugin/core/deepseek_client.py`
   - 不能在 `deepseek_tcad_plugin/src/core/` 下

3. **图标必须是 GIF 格式**
   - SWB 不支持 JPG/PNG 作为工具图标
   - 推荐尺寸：64x64
   - 使用 Pillow 转换：`python -c "from PIL import Image; Image.open('deepseek.jpg').resize((64,64)).save('deepseek.gif')"`

4. **tooldb_tcad 必须放在 `/home/tcad/STDB/` 目录下**
   - 这是 SWB 读取用户工具的标准位置
   - 文件名必须是 `tooldb_tcad`

5. **节点启动器必须在SWB项目目录中**
   - 每个项目需要一个 `deepseek_tcad_ai.py`
   - 路径要与 tooldb_tcad 中 `input,commands,file` 属性匹配

## 四、在SWB中使用

### 4.1 启动SWB

```bash
swb &
```

### 4.2 打开项目

菜单：`File > Open Project` → 选择你的Sentaurus项目目录（包含 `deepseek_tcad_ai.py` 的那个目录）

### 4.3 重新加载配置

菜单：`Project > Reload`

### 4.4 添加DeepSeek工具节点

1. 点击工具栏 `+` 按钮（或菜单 `Edit > Add Tool`）
2. 在弹出的对话框中：
   - **Tool**：从下拉列表选择 `deepseek_tcad`
   - **Label**：输入 `ai`（小写）
   - **Command line**：留空
   - **Comment**：留空
   - **Scenario**：`default`
3. 点击 `OK`

### 4.5 运行工具

1. 在项目树中选中 `ai` 节点
2. 点击 `Run`（绿色播放按钮 ▶）
3. 或右键 → Run
4. 双击 `ai` 节点可以编辑命令文本（可选）

GUI窗口会自动弹出。

## 七、常见问题

### Q1: 工具显示为 unavailable？

检查清单：
```bash
# 1. Python是否存在且可执行
ls -la /home/tcad/work/deepseek_tcad_plugin/.venv/bin/python
/home/tcad/work/deepseek_tcad_plugin/.venv/bin/python --version

# 2. tooldb_tcad是否存在且内容正确
cat /home/tcad/STDB/tooldb_tcad

# 3. 图标是否存在
ls -la /usr/synopsys/sentaurus/*/tcad/current/lib/glib2/icons/deepseek.gif

# 4. 清除SWB缓存
cd /path/to/your/swb/project
rm -rf .database .history .status
```

最后**完全关闭SWB并重新打开**。

### Q2: GUI窗口不出现？

```bash
# 1. 检查DISPLAY环境变量
echo $DISPLAY  # 应该输出 :0 或 :0.0

# 2. 手动测试GUI启动
/home/tcad/work/deepseek_tcad_plugin/.venv/bin/python \
    /home/tcad/work/deepseek_tcad_plugin/src/main.py

# 3. 检查模块导入是否正确
# 如果报错 ModuleNotFoundError: No module named 'core'
# 说明 sys.path 设置有问题，检查 main.py 中的 PLUGIN_DIR 计算
```

### Q3: API调用失败？

```bash
# 检查配置文件中的API Key
cat /home/tcad/work/deepseek_tcad_plugin/config/settings.ini

# 测试API连通性
/home/tcad/work/deepseek_tcad_plugin/.venv/bin/python -c "
from core.deepseek_client import DeepSeekClient
import configparser, os
cfg = configparser.ConfigParser()
cfg.read(os.path.expanduser('~/work/deepseek_tcad_plugin/config/settings.ini'))
client = DeepSeekClient(cfg.get('deepseek','api_key'))
print(client.chat_completion([{'role':'user','content':'Hi'}]))
"
```

### Q4: 图标不显示？

```bash
# 1. 检查图标格式
file /usr/synopsys/sentaurus/*/tcad/current/lib/glib2/icons/deepseek.gif
# 应该显示 GIF image

sudo cp /home/tcad/work/deepseek_tcad_plugin/icons/deepseek.gif \
```

### Q5: 如何迁移到另一台机器？

1. 复制整个 `deepseek_tcad_plugin` 目录到新机器
2. 在新机器上运行 `bash install.sh`（或按方式二手动安装）
3. 脚本会重新创建虚拟环境和注册工具
4. **不需要**重新复制图标以外的代码

### Q6: 多个SWB项目如何使用同一个插件？

每个SWB项目只需要在该项目目录下创建一个 `deepseek_tcad_ai.py` 节点启动器：

```bash
cd /path/to/another/swb/project
cat > deepseek_tcad_ai.py << EOF
#!/home/tcad/work/deepseek_tcad_plugin/.venv/bin/python
import sys, os, subprocess
subprocess.Popen([sys.executable, "/home/tcad/work/deepseek_tcad_plugin/src/main.py", "--project", os.getcwd()],
                 stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL, start_new_session=True)
print("DeepSeek AI Assistant launched!")
EOF
chmod +x deepseek_tcad_ai.py
```
